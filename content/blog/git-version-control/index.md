## Что такое управление версиями

Управление версиями (version control) — это система, которая записывает изменения в файлах с течением времени, позволяя вернуться к любой предыдущей версии. Это незаменимый инструмент для разработчиков, учёных и всех, кто работает с текстовыми файлами.

### Зачем нужно управление версиями

Системы контроля версий решают несколько важных задач:

- **История изменений** — можно отследить, кто, когда и что изменил в проекте.
- **Откат изменений** — если что-то сломалось, легко вернуться к рабочей версии.
- **Совместная работа** — несколько человек могут одновременно работать над одним проектом без конфликтов.
- **Ветвление** — можно экспериментировать с кодом в отдельной ветке, не затрагивая основную версию.

## Типы систем контроля версий

### Локальные

Простейший подход — копировать файлы в другую директорию. Более продвинутый вариант — локальные VCS вроде RCS, которые хранят набор патчей между версиями.

### Централизованные (CVCS)

Системы вроде CVS, Subversion (SVN) и Perforce используют единый сервер, на котором хранится вся история. Клиенты получают файлы из этого центрального хранилища.

### Распределённые (DVCS)

Git, Mercurial и Bazaar — распределённые системы. Каждый разработчик имеет полную копию репозитория, включая всю историю. Это обеспечивает надёжность и позволяет работать офлайн.

## Git — самая популярная VCS

Git был создан Линусом Торвальдсом в 2005 году для разработки ядра Linux. Сегодня Git является стандартом де-факто в мире разработки программного обеспечения.

### Основные концепции Git

- **Репозиторий (repository)** — хранилище файлов и истории их изменений.
- **Коммит (commit)** — снимок состояния проекта в определённый момент времени.
- **Ветка (branch)** — независимая линия разработки.
- **Слияние (merge)** — объединение изменений из разных веток.
- **Удалённый репозиторий (remote)** — копия репозитория на сервере (например, на GitHub).

### Основные команды Git

```bash
# Инициализация нового репозитория
git init

# Клонирование существующего репозитория
git clone https://github.com/user/repo.git

# Проверка статуса файлов
git status

# Добавление файлов в индекс
git add filename.txt
git add .

# Создание коммита
git commit -m "Описание изменений"

# Отправка изменений на удалённый сервер
git push origin main

# Получение изменений с удалённого сервера
git pull origin main
```

### Рабочий процесс с Git

Типичный рабочий процесс с Git выглядит так:

1. Создаём или клонируем репозиторий.
2. Создаём новую ветку для задачи.
3. Вносим изменения в файлы.
4. Добавляем изменения в индекс (`git add`).
5. Фиксируем изменения коммитом (`git commit`).
6. Отправляем ветку на сервер (`git push`).
7. Создаём Pull Request для ревью.
8. После одобрения сливаем ветку в основную.

## GitHub — платформа для совместной работы

GitHub — это веб-платформа для хостинга Git-репозиториев, которая предоставляет дополнительные инструменты для совместной разработки: Pull Requests, Issues, GitHub Actions, GitHub Pages и другие.

В рамках нашего индивидуального проекта мы используем GitHub для хостинга исходного кода сайта и GitHub Pages для его публикации.

## Заключение

Git — мощный инструмент для управления версиями, который необходим каждому разработчику. Понимание его основ поможет эффективно работать над проектами любой сложности, как индивидуально, так и в команде.
